模块一：评分引擎

三维度加权 · 主张验证 65% + 域名权威 20% + 内容质量 15%

📋

主张验证 (65%)

得分 = 0.45 + 0.5 × (支持 − 反驳)

证据强度 = relevance × authority × confidence

top-3 指数衰减加权 · 多源互证加分

core 被高权威反驳 → 封顶 0.40

🌐

域名权威 (20%)

.gov / .edu → 0.95

Reuters / 新华社 → 0.90

普通网页 → 0.50

.top / .xyz → 0.10

authority_sort 自动评分

📝

内容质量 (15%)

满分 1.0，只扣分：

内部矛盾 −0.30 封顶

模糊表述密度 −0.30 封顶

提示注入 −0.45 封顶

绝对化用语 −0.15 封顶

评分实例

云南旅游诈骗页 · ss.ynsjly.top

strength = relevance × authority × confidence

support = top3(指数衰减: 1.0, 0.6, 0.36)
contradict = top3(同上)

得分 = 0.45 + 0.5×(support − contradict)
多源 bonus = 1 + 0.08×log₂(n_domains)
维度得分原因
主张验证0.355 条均无法验证
域名权威0.10.top TLD 黑名单
内容质量0.20广告 + 矛盾 + 注入
综合0.28Low Credibility
0.65×0.35 + 0.20×0.10 + 0.15×0.20 = 0.2775

模块二：上下文压缩

⚙️

策略

  • Token 估算：字符数 / 4
  • 超预算自动触发压缩
  • 保留最近 K 轮原文对话
  • 早期对话压缩为 system 备忘
  • Tool 结果过长先截断再注入
📝

摘要保留内容

1. 任务目标
2. 已完成的步骤和输出
　 (claims / 证据 / 判定结论)
3. 已使用的工具和结果
4. 当前进度（下一步做什么）
不遗漏关键事实或已验证结论
